NearBy - AI-Powered People Discovery at Events
================================================
INIT 2026 Hackathon Project
Date: March 7, 2026


THE PROBLEM
-----------
At events, conferences, and festivals you're surrounded by interesting people
but have no way to discover WHO is actually around you. You might be standing
next to the exact person you've been looking for - a co-founder, a mentor,
someone in your niche field - and never know it.

Traditional networking is random. LinkedIn requires you to already know someone.
Event apps require manual profile creation that nobody fills out properly.


THE IDEA
--------
An app that uses AI web-scraping to automatically build rich professional
profiles of event attendees - no profile creation needed.

A person joins by providing just their name and email (+ optional brief bio or CV).
They accept terms consenting to a deep public internet search about them.
The AI then scrapes and synthesizes everything publicly available about that person:
career history, achievements, education, interests, goals, publications, projects.

Once everyone at an event is indexed, you can:
- See all people physically nearby (GPS-based)
- Search by keywords: "find all astronauts", "machine learning researchers",
  "people from Tartu University", "founders looking for co-founders"
- Scan someone's QR code to instantly see their AI-generated profile
- Discover unexpected connections and shared interests

Example use case (Joosep's TalveFestival):
  "Show me everyone here who has worked in space tech"
  -> Returns 3 people within 200m with full career context


HOW IT WORKS (Technical)
------------------------
1. USER JOINS
   - Enters: name, email, optional bio/CV upload
   - Accepts terms: consent for public internet search
   - Backend triggers AI-powered web research

2. AI DEEP SEARCH (Perplexity API - sonar-pro model)
   - Searches LinkedIn, personal sites, GitHub, publications,
     news articles, university pages, social media
   - Returns structured profile: career, education, achievements,
     current role, interests, goals, tags
   - All from PUBLIC information only

3. DISCOVERY
   - GPS/Geolocation: find people physically nearby at the event
   - Keyword search: filter all attendees by career fields, skills, goals
   - QR code: each person gets a unique QR linking to their profile

4. STORAGE
   - Profiles stored per-event (not permanent)
   - User can request deletion at any time
   - Data only persists for the duration of the event


TECH STACK (Prototype)
----------------------
- Backend:  Python FastAPI (single file, minimal)
- Frontend: Plain HTML/CSS/JS (single-page app, no framework)
- Database: SQLite (local, simple)
- AI:       Perplexity API (sonar-pro) for web research
- QR:       python-qrcode library
- Location: Browser Geolocation API
- Hosting:  Local / can deploy to any VPS or Vercel


LEGAL / PRIVACY CONSIDERATIONS
------------------------------
- Only PUBLIC information is scraped - nothing private
- User must explicitly accept terms before their search runs
- The AI search is the equivalent of someone Googling you - just automated
- GDPR-compliant approach: consent-based, right to deletion
- Event-scoped data: profiles are tied to events, not stored permanently
- No data sold or shared with third parties
- User can opt out and delete their profile at any time
